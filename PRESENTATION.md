> "저희는 이 문제를 **자연어 한 줄**로 해결했습니다."

## [0:30 ~ 1:30] 제품 소개

**[화면: 대시보드 메인 화면 띄우기]**

> "AI Container Orchestrator는 자연어로 컨테이너 인프라를 제어하는 오케스트레이션 플랫폼입니다."

**핵심 기능 설명 (짧게):**

> "화면을 보시면, 이 대시보드 하나로 모든 것을 관리합니다."
>
> - "**멀티노드 클러스터 관리** - 여러 서버를 하나의 화면에서 봅니다."
> - "**AI 채팅 어시스턴트** - 'nginx 3개 배포해줘'라고 말하면 됩니다."
> - "**실시간 리소스 모니터링** - CPU, 메모리, 네트워크 I/O를 컨테이너 단위로 추적합니다."
> - "**드래그앤드롭 컨테이너 이동** - 마우스로 끌어서 노드 간 마이그레이션합니다."
>
> "Kubernetes 없이도 멀티노드 오케스트레이션이 가능합니다. 복잡한 YAML 작성 필요 없습니다."

## [1:30 ~ 3:00] 라이브 시연 (핵심!)

**[화면: 대시보드 라이브]**

### 시연 1: 클러스터 상태 확인

> "먼저 클러스터 상태를 확인합니다. 3개 노드가 모두 Healthy 상태입니다."

**[화면: 노드 카드 3개 - CPU/MEM/NET 실시간 수치 표시]**

### 시연 2: AI 자연어 배포

> "AI 채팅 어시스턴트를 열겠습니다."

**[채팅창에 입력: "nginx 3개 배포해줘"]**

> "입력했습니다. AI가 명령을 해석하고... 자동으로 3개 노드에 분산 배포합니다."

**[화면: 배치 현황 그리드에서 nginx 컨테이너 3개가 나타남]**

> "배치 현황을 보시면 nginx가 각 노드에 하나씩 분산된 것이 보입니다. 로드밸런싱까지 자동입니다."

### 시연 3: 드래그앤드롭 마이그레이션

> "이제 컨테이너를 다른 노드로 옮겨보겠습니다."

**[마우스로 컨테이너를 드래그하여 다른 노드로 이동]**

> "이렇게 드래그앤드롭으로 라이브 마이그레이션이 됩니다. CLI 명령어 한 줄 필요 없습니다."

### 시연 4: 블록체인 원클릭 배포

> "마지막으로 블록체인 네트워크를 원클릭으로 배포합니다."

**[화면: QuickStart 메뉴에서 블록체인 배포 클릭]**

> "QuickStart에서 블록체인을 선택하면, 전체 네트워크가 자동으로 구성됩니다."

## [3:00 ~ 4:15] AX 전환 노하우 (25점 - 최고 배점!)

**[화면: 개발 과정 요약 슬라이드 또는 Claude Code 화면]**

> "이제 가장 중요한 부분, AI를 어떻게 활용했는지 말씀드리겠습니다."
>
> "이 프로젝트는 **Claude Code**를 전면 활용하여 개발했습니다."

### 활용 방법 1: 아키텍처 설계

> "처음에 Claude에게 '컨테이너 오케스트레이터를 설계해줘, 멀티노드 지원, 자연어 제어 포함'이라고 요청했습니다."
>
> "AI가 전체 시스템 구조를 제안했고, 오케스트레이터 / 워커 / NL엔진 / 모니터링 4개 모듈 구조가 나왔습니다. 이걸 기반으로 개발을 시작했습니다."

### 활용 방법 2: 모듈별 구현

> "각 모듈을 Claude Code에게 맥락과 함께 요청했습니다."
>
> - "'NL 엔진 구현해줘, 한국어 자연어 명령을 Docker API 호출로 변환' - 코드가 생성됐습니다."
> - "'실시간 리소스 모니터링 구현해줘, CPU/메모리/네트워크 I/O 포함' - 모니터링 모듈이 나왔습니다."
> - "'드래그앤드롭 컨테이너 마이그레이션 구현해줘' - 프론트엔드와 백엔드 코드가 동시에 생성됐습니다."

### 활용 방법 3: 언어 포팅

> "초기에 Python으로 프로토타입을 만들고, 성능을 위해 Go로 포팅했습니다."
>
> "'Python 코드를 Go로 변환해줘' - **Claude Code가 자동으로 변환**했습니다. 수동 포팅이었으면 며칠 걸렸을 작업입니다."

### 활용 방법 4: 해커톤 당일 추가 기능

> "해커톤 당일에도 Claude Code로 추가 기능을 실시간 구현했습니다."
>
> - "AI 채팅 어시스턴트 UI 추가"
> - "자동 복구 기능 추가"
> - "컨테이너별 리소스 모니터링 추가"

### 핵심 인사이트

> "저희가 얻은 가장 중요한 교훈은 이것입니다."
>
> "**AI에게 좋은 맥락을 제공하면, 복잡한 시스템도 빠르게 구현할 수 있습니다.**"
>
> "전체 코드 10,000줄 이상을 AI와 함께 작성했고, 기존에 수 주 걸릴 개발을 단축했습니다. AI는 도구가 아니라 **팀원**이었습니다."

## 발표 준비 체크리스트

- [ ] Docker 환경 사전 구동 (3노드 클러스터 + 워커)
- [ ] 대시보드 접속 확인 (브라우저 탭 미리 열기)
- [ ] AI 채팅 기능 동작 테스트
- [ ] 시연용 컨테이너 정리 (깨끗한 상태에서 시작)
- [ ] QuickStart 블록체인 배포 테스트
- [ ] 네트워크 상태 확인 (시연 중 끊기면 안 됨)
- [ ] 백업 시나리오 준비 (시연 실패 시 녹화 영상)

## 발표 팁

- **시연에서 멈추면 안 됨** - 반드시 사전에 환경을 완벽히 준비할 것
- **AI 채팅 기능을 핵심 시연으로** - 자연어 입력 → 실행 결과가 시각적 임팩트가 가장 큼
- **AX 노하우는 구체적 숫자와 경험으로** - "10,000줄", "수 주 → 단축", "해커톤 당일 추가 구현" 등
- **속도 조절** - 시연 구간(1:30)이 가장 길므로 여기서 천천히, 나머지는 리듬감 있게
- **자신감 있게** - 실제 작동하는 제품이라는 점을 강조
